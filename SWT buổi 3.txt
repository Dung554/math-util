1. DEV 
- Dev phải có trách nhiệm test code của mình trc khi bàn giao cho ngkhac lắp ghép và sử dung, hoặc chính mình sử dung
* Test code của mình là test cái gì và như thế nào?
- trong 1 dự án phần mềm, đơn vị nhỏ nhất trong src code là hàm/method/class hay còn gọi là unit of code
- Việc ktra từng hàm chạy đúng hay sai, từng class chứa các hàm đúng hay sai. Đây là việc của Dev, ai code thì người đó hiểu code nhất -> Unit testing

* Làm sao để test 1 hàm?
- gọi hàm thử, đưa tham số và in ra màn hình thử, in ra cửa sổ console/terminal
- gọi hàm, đưa tham số, in ra file .log (tomcat có thư mục logs) chứa nhật kí hoạt động của cái app. class logger
